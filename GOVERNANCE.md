# ROFS-8 Governance Model

**Project:** Sovereign Logic Systems (SLS)  
**Authority:** Reco Fu / Lead Architect  

## 1. Purpose
This document defines how decisions are made, how contributions are accepted, and how the ROFS-8 standard evolves under Sovereign Logic Systems.

## 2. Roles & Responsibilities
- **Maintainer / Benevolent Dictator:** Responsible for architectural consistency, final versioning, and merging pull requests. The maintainer ensures the core philosophy (precision over hype, evidence-backed evolution) is preserved.
- **Contributors:** Community members who submit issues, pull requests, test cases, or reference implementations.

## 3. Contribution Workflow
1. **Open an Issue:** Discuss major conceptual changes or new reference implementations before submitting heavy pull requests.
2. **Pull Requests (PR):** Must pass consistency checks and adhere to the terminology boundary (using `conceptual` / `proposed` where applicable).
3. **Compatibility:** Derivative works or compatible tools must explicitly state which version of ROFS-8 they implement (see Whitebook Compatibility Notes).

## 4. Trademark & Naming Policy
ROFS-8 is published under the Apache 2.0 license. The name "ROFS-8" and "Sovereign Logic Systems" remain identifiers of the canonical origin. Derivative implementations must avoid misleading branding that implies official SLS certification unless explicitly authorized.
